## КРИТИЧЕСКИЕ БАГИ (исправлено в этом аудите)

### 1. WebSocket URL — неверный путь `/ws/recalculate` вместо `/api/ws/recalculate`

**Файлы:** `frontend/src/api/subsidenceSocket.ts`, `frontend/vite.config.ts`

Бэкенд регистрирует WebSocket эндпоинт с префиксом `/api`:
```python
# main.py
app.include_router(subsidence_router, prefix="/api")
# subsidence.py
@router.websocket('/ws/recalculate')  # → итого /api/ws/recalculate
```

Фронтенд строил URL без `/api`:
```ts
// ДО: wsUrl() → ws://host/ws/recalculate  (404 — не существует)
// ПОСЛЕ: wsUrl() → ws://host/api/ws/recalculate  (корректно)
```

Vite proxy имел отдельную запись `/ws` → `ws://localhost:8000`, которая отправляла запросы на несуществующий путь. Исправлено добавлением `ws: true` к основному прокси `/api`.

**Последствие:** Расчёт субсиденции никогда не работал в dev-режиме — WebSocket соединение не устанавливалось, `isComputing` зависало в `true`.

### 2. Race condition в `sendRecalculation` при CONNECTING-состоянии сокета

**Файл:** `frontend/src/api/subsidenceSocket.ts`

Старый код перезаписывал `socket.onopen` напрямую, обрывая backoff-reset handler. Новый код использует модуль-уровневый `pendingMessages: string[]` — сокет сбрасывает очередь при открытии. Лишние сообщения отбрасываются (`pendingMessages.length = 0` перед push) — отправляется только последний запрос.

### 3. Утечка памяти в `useFormationDrag` — неочищенная подписка

**Файл:** `frontend/src/hooks/useFormationDrag.ts`


## ВЫСОКИЙ ПРИОРИТЕТ (не исправлено, требует внимания)

### 4. `pendingDepthPatches` — модуль-уровневый Map, не очищается при смене скважины

**Файл:** `frontend/src/stores/wellDataStore.ts`

`pendingDepthPatches` объявлен вне store как `const pendingDepthPatches = new Map<string, ReturnType<typeof setTimeout>>()`. При переключении на другую скважину таймеры от предыдущей скважины остаются активными. Если таймаут сработает — он отправит PATCH с `formation.id` из старой скважины в контексте новой.

**Риск:** Silent data corruption при быстром переключении скважин во время редактирования отбивок.

**Рекомендация:** Очищать все таймеры в `wellDataStore.reset()`.

### 5. `wellDataStore.ts` — parquet читается только по первому URI

**Файл:** `app/src/subsidence/api/wells.py`, функция `load_curves_from_parquet`

```python
data = pd.read_parquet(curve_rows[0].data_uri, ...)
```

Если кривые скважины хранятся в нескольких parquet-файлах (разные импорты), все кроме первого файла будут проигнорированы без ошибки.

### 6. `get_deviation` в `wells.py` — чтение parquet вне сессии

**Файл:** `app/src/subsidence/api/wells.py`

```python
with session:
    survey_row = session.scalar(...)  # запрос в сессии
```

Само по себе не вызывает ошибку (parquet читается из файла, не из ORM), но если SQLAlchemy lazy-load атрибутов `survey_row` произойдёт после закрытия сессии — DetachedInstanceError. Нужно явно читать `data_uri` внутри `with`.

### 7. `isComputing` зависает в `true` при обрыве WebSocket

**Файл:** `frontend/src/stores/computedStore.ts`

Если WS-соединение разрывается после отправки запроса (до получения ответа), `isComputing` остаётся `true` навсегда — `setResults` и `clearResults` никогда не вызываются. Нет timeout/watchdog.

**Рекомендация:** Установить timeout в `triggerRecalculation`:
```ts
let computeTimeout: ReturnType<typeof setTimeout> | null = null
triggerRecalculation() {
  if (computeTimeout) clearTimeout(computeTimeout)
  computeTimeout = setTimeout(() => set({ isComputing: false, computeError: 'Timeout' }), 30_000)
  ...
}
```

### 10. `_autosave_loop` в `ProjectManager` — синхронный IO в async контексте

**Файл:** `app/src/subsidence/data/project_manager.py`

Если `autosave()` вызывается напрямую в async-методе (не через `run_in_executor`), SQLite-запись блокирует event loop. Нужно:
```python
await asyncio.get_event_loop().run_in_executor(None, self.autosave)
```

### 11. `activate_strat_chart` обходит undo-систему

**Файл:** `app/src/subsidence/api/strat_chart.py`

Активация чарта вызывает `mark_dirty()` напрямую минуя `UndoStack`. Это нарушает инвариант undo/redo: после активации чарта нельзя откатить это действие. Все мутирующие операции должны идти через `UndoCommand`.

```python
# Phase 5: tectonic_subsidence = total_col * (RHO_MANTLE - rho_s_avg) / (RHO_MANTLE - RHO_WATER) + water_depth_m
```

### 14. Неверный комментарий в `backstrip.py`

**Файл:** `app/src/subsidence/data/backstrip.py`, строка 171

```python
for i in reversed(active_indices):  # oldest (deepest) first  ← НЕВЕРНО
```

`active_indices` сортированы по возрастанию (index 0 = самый старый, deepest). `reversed()` даёт youngest first. Логика правильная (молодой = ближе к поверхности = z_top=0), комментарий ложный. Исправить на:

```python
for i in reversed(active_indices):  # youngest first — sits at surface (z_top = 0)
```
